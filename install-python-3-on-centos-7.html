centos7 编译安装 Python3.6

published at: 2018-10-15T16:56:00+08:00

centos 7 安装 Python3

前言

大多数 Linux 发行版都自带Python2.7。不过Python2.7会逐渐被淘汰，不如直接用Python3。

你也许会想要把Python2版本给删掉，但我建议不要这样做，因为有程序依赖现有的Python2环境，把Python2删除后会报各种错误。

我们可以编译安装Python3，使得Python2和Python3这两个版本并存，同时又互不干扰，你可以随时使用任意一个版本的Python。这样的话，即不破坏当前的Python2的环境，又能使用Python3，好处多多。

Python3也有很多不同的版本，去Python的官网 https://www.python.org/downloads/source/ 下载你想要安装的版本。建议用Python3.6.6这个版本。

安装Python3

安装依赖环境

首先要安装一些Python3的依赖包。不装的话后面会报各种错误。

# yum -y install zlib-devel bzip2-devel openssl-devel ncurses-devel sqlite-devel readline-devel tk-devel gdbm-devel db4-devel libpcap-devel xz-devel

下载Python3.6

在Python官网找到Python3.6.6的下载链接，下载到本机上。

wget https://www.python.org/ftp/python/3.6.6/Python-3.6.6.tgz

创建Python3.6版本的安装目录

# mkdir -p /usr/local/python3

将下载下来的安装包解压缩

# tar -zxvf Python-3.6.6.tgz

进入解压后的目录，编译安装。

 cd Python-3.6.6

 ./configure --prefix=/usr/local/python3

 make 

 make install

注意make编译可能会花一点时间，不过不会太久。

建立python3的软链

# ln -s /usr/local/python3/bin/python3 /usr/bin/python3

将/usr/local/python3/bin加入你的环境变量PATH

用vim编辑器打开并编辑

vim ~/.bash_profile

找到PATH=$PATH:$HOME/bin这一行，在这一行的末尾附加上你的Python3的命令路径，如下：

PATH=$PATH:$HOME/bin:/usr/local/python3/bin

保存退出。

然后使命令生效。

# source ~/.bash_profile

此刻，你的Python3就安装成功了！

使用Python3

在终端输入Python3，你会看到：

Python 3.6.6 (default, Sep  9 2018, 03:38:26) 
[GCC 4.8.5 20150623 (Red Hat 4.8.5-28)] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>

按下ctr+d可以退出，或者输入exit()，然后回车，也能退出。

在终端输入Python，你会看到：

Python 2.7.5 (default, Jul 13 2018, 13:06:57) 
[GCC 4.8.5 20150623 (Red Hat 4.8.5-28)] on linux2
Type "help", "copyright", "credits" or "license" for more information.
>>>

可以看到，你的Python2和Python3二者并存。

安装Python3.6后，你可能也同时安装了pip工具。如果你之前没有安装过pip，那以后在使用pip下载时，可以直接用pip命令，例如：

pip install you-get

但如果你之前已经安装了pip，那你就需要指明是python3版本的pip：

pip3 install you-get
